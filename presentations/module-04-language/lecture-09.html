Проблема

Компьютер понимает только числа

Текст для компьютера — просто набор символов без смысла.

Вопрос

Как превратить «кошка сидит на коврике» в числа, которые сохраняют смысл?

Без ответа на этот вопрос не было бы ни ChatGPT, ни переводчиков, ни голосовых помощников.

Секция 1. Токенизация

Токен — не всегда слово

Примеры разбиения
«I love cats» → I love cats — 3 токена = 3 слова
«Невероятный» → Не вероят ный — 3 токена, 1 слово
«Достопримечательность» → Дост опре дел енности — 4 токена, 1 слово

Частые слова = один токен. Редкие и длинные = разбиваются на подслова.

Секция 1. Токенизация

BPE — Byte Pair Encoding

Самый популярный алгоритм токенизации:

1. Начинаем с символов: «к», «о», «т»
2. Находим самую частую пару: «к», «о» → «ко»
3. Добавляем «ко» в словарь
4. Повторяем, пока словарь не наберёт нужный размер

GPT-4: словарь ~100 000 токенов. Из них можно «собрать» любой текст.

Секция 1. Токенизация

Русский vs Английский

🇷🇺 Русский

«Искусственный интеллект меняет мир»

Искусственный интеллект меняет мир

7 токенов

🇬🇧 English

«Artificial intelligence changes the world»

Artificial intelligence changes the world

6 токенов

Русский текст обычно даёт больше токенов — слова длиннее, морфология сложнее. Для ChatGPT русский текст «дороже».

Секция 2. Embeddings

Проблема: как представить слова?

Просто пронумеровать? «кот» = 142, «кошка» = 893?

One-hot encoding

«кот» → [0, 0, ..., 1, ..., 0, 0] — единица на позиции 142

«кошка» → [0, 0, ..., 0, 1, ..., 0] — единица на позиции 893

✗ Все слова одинаково «далеки» друг от друга

✗ Вектор из 100 000 чисел — для одного слова

✗ Не масштабируется

Секция 2. Embeddings

Word2Vec — прорыв 2013

«Слово характеризуется его окружением»

Каждое слово → компактный вектор из 100-300 чисел. Слова в похожем контексте = близкие вектора.

Как обучается

1. Берём миллиарды слов текста
2. Для каждого слова смотрим соседей
3. Корректируем вектор, чтобы предсказывать соседей
4. После миллионов итераций — вектора отражают смысл

Результат

Животные рядом: кот, собака, хомяк
Страны рядом: Франция, Россия, Япония
Глаголы рядом: бежать, идти, ползти

Секция 2. Embeddings

Магия аналогий

Вектора поддерживают арифметику смысла:

король − мужчина + женщина ≈ королева
Париж − Франция + Россия ≈ Москва
врач − женщина + мужчина ≈ ??? (предсказание модели)

Это не запрограммировано — модель сама «открыла» отношения, анализируя тексты.

Секция 2. Embeddings

Визуализация: t-SNE

Сжимаем 300 измерений в 2 для графика:

🐾 Животные

кот → собака → хомяк → попугай

🌍 Страны

Франция → Россия → Япония → Бразилия

🏃 Движение

бежать → идти → ползти → лететь

Похожие слова — рядом. Разные — далеко. Это и есть «понимание» для компьютера.

Секция 2. Embeddings

Ограничения Word2Vec

Одно слово = один вектор

«Банк» (финансовый) и «банк» (река) — одинаковый вектор. Контекст не учитывается.

Статический

После обучения вектора фиксированы. Не адаптируются к контексту предложения.

Решение — впереди

Трансформеры и BERT решат эту проблему — контекстные embeddings. Лекция 10-11.

Секция 3. Порядок слов

Порядок слов имеет значение

Предложение 1
«Собака кусает человека»
≠
Предложение 2
«Человек кусает собаку»

Те же три слова. Тот же «мешок». Абсолютно разный смысл.

Ещё пример

«Я не сказал, что он украл деньги»

«Я сказал, что он не украл деньги»

Разница в одно слово — разница между обвинением и защитой.

Секция 3. Порядок слов

Bag of Words — плохо

Мешок слов теряет порядок
«кот съел рыбу» → {кот:1, съел:1, рыба:1}
«рыба съела кота» → {рыба:1, съел:1, кот:1}

Для компьютера — одно и то же!

✗ Порядок теряется полностью

✗ Длинные зависимости не ловятся

✗ n-граммы — частичное решение, комбинаторный взрыв

Секция 3. Порядок слов

Что нам нужно?

✓ Знает «смысл» каждого слова — Word2Vec ✓

✓ Учитывает порядок слов — bag of words ✗, n-граммы ~

✓ Учитывает контекст слова в предложении — пока нет ✗

✓ Обрабатывает тексты любой длины — пока нет ✗

Ответ

Этот метод — трансформер. Следующая лекция.
