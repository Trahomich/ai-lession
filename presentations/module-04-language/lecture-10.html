Проблема

Почему RNN не справлялся

RNN: читает слово за словом

Проблема 1: короткая память — забывает начало текста

Проблема 2: нет параллелизма — каждое слово ждёт предыдущее

Слово 1 → Слово 2 → Слово 3 → ... → Слово 100
       ↓         ↓         ↓                  ↓
h₁      →   h₂    →   h₃    → ... → h₁₀₀

Градиент затухает — информация теряется
Проблема

LSTM и GRU — полумеры

LSTM добавила «вентили» — что запомнить, что забыть

GRU — упрощённая версия LSTM

Частично решили проблему памяти

Но остались последовательными — параллелить нельзя

Для GPU — которые идеально подходят для параллельных вычислений — последовательная обработка была катастрофой.

К 2016 году стало ясно: нужен принципиально другой подход

Ключевая идея

Пусть каждое слово «посмотрит» на все остальные

Банк закрылся после того как река затопила берег

«банк» ←сильная связь→ «река», «затопила»
Это берег реки, а не финансовое учреждение!

Контекст определяет смысл

Attention

Query, Key, Value

Аналогия с библиотекой:


Query (запрос) — «что я ищу» → вы говорите библиотекарю «книги о кошках»

Key (ключ) — «что я предлагаю» → карточка каждой книги с ключевыми словами

Value (значение) — «содержание» → сам текст книги

Attention

Как работает Attention

1. Каждое слово → Q, K, V (три вектора)

2. Q × Kᵀ → оценка «релевантности» каждой пары слов

3. Softmax → нормализация в вероятности (сумма = 1)

4. Взвешенная сумма V → итоговый вектор слова

Attention(Q,K,V) = softmax(Q × Kᵀ / √d) × V

Деление на √d — масштабирование, чтобы оценки не «взрывались»

Attention

Self-attention в действии

«Программист написал код, потому что он хотел протестировать его»

«он» ← сильный attention → «программист»

«его» ← сильный attention → «код»


Никто не прописывал эти правила — модель выучила их на миллиардах текстов

Attention

Multi-head attention — много голов

Голова 1: синтаксис
Голова 2: семантика
Голова 3: позиция
Голова 4: тема

Каждая голова смотрит «со своей колокольни»

Результаты конкатенируются → линейное преобразование

Позиция

Positional encoding — где стоит слово

Self-attention обрабатывает все слова одновременно

Модель не знает порядок — нужно добавить!

Embedding слова + Позиционный код = Итоговый вектор

[0.2, 0.5, ...] + [0.01, 0.84, ...] = [0.21, 1.34, ...]
слово         позиция 3       итог
Аналогия: в кинотеатре все зрители одинаковые, но у каждого билет с номером ряда и места. Без билета — хаос.
Архитектура

Encoder vs Decoder

Encoder: понимает текст
BERT, поиск, классификация
Decoder: генерирует текст
GPT, перевод, диалог
Оба: текст → текст
T5, BART, перевод

GPT = только Decoder — читает предыдущие токены, предсказывает следующий

BERT = только Encoder — читает весь текст, создаёт представления

Архитектура

Маскирование в Decoder

Decoder генерирует последовательно, но attention — параллельный

Нельзя «подглядывать» в будущие токены!

Токен 1: видит [1 0 0 0 0]
Токен 2: видит [1 1 0 0 0]
Токен 3: видит [1 1 1 0 0]
Токен 4: видит [1 1 1 1 0]
Токен 5: видит [1 1 1 1 1]

Треугольная маска — будущее закрыто

Итоги

Что запомнить

  • Attention — каждое слово «видит» весь контекст сразу
  • Positional encoding — сообщает, где стоит слово
  • Multi-head — несколько перспектив одновременно
  • Skip connections — информация перепрыгивает через слои
  • Параллелизм — быстрое обучение на гигантских данных
  • GPT = Decoder, BERT = Encoder, T5 = оба
